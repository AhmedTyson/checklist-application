2. The Git & GitHub Learning Roadmap

Once you finish the video, follow this roadmap to solidify your skills step-by-step:

Phase 1: The Local Fundamentals
  • Concepts: Working Directory vs. Staging Area vs. Repository.
  • Actions: Initializing a repository, checking status, staging files, and making your first commits.
  • Goal: Be able to track your own solo project locally.
Phase 2: Time Travel (Undoing Mistakes)
  • Concepts: The HEAD pointer and commit hashes.
  • Actions: Viewing history, discarding uncommitted changes, and reverting to older versions.
  • Goal: Feel confident that you can't "break" your code permanently.
Phase 3: Branching & Merging
  • Concepts: Parallel development.
  • Actions: Creating branches for new features, switching branches, merging features back into the main codebase, and handling merge conflicts.
  • Goal: Develop features safely without breaking the main working code.
Phase 4: Remote Repositories (GitHub)
  • Concepts: Local vs. Remote, Authentication (SSH/Tokens).
  • Actions: Pushing local code to GitHub, cloning existing repositories, and pulling updates from colleagues.
  • Goal: Back up your code to the cloud and prepare for collaboration.
Phase 5: Collaboration & Open Source
  • Concepts: GitHub Flow, Forks, and Pull Requests.
  • Actions: Forking someone else's repo, pushing changes to your fork, and opening a Pull Request for review.
  • Goal: Contribute to open-source projects or work smoothly on a corporate team.

4. How to Commit Using Graphical Interfaces (No CMD)

You do not have to use the terminal to be great at Git! Modern IDEs have excellent built-in Git tools.

How to Commit in Visual Studio Code (VS Code)

  1. Open the Source Control Tab: On the far left sidebar, click the icon that looks like a tree branch with nodes (or press Ctrl+Shift+G).
  2. Stage Your Changes: Hover over the files you changed under the "Changes" list. Click the + (plus icon) next to the file to move it to "Staged Changes" (this is the equivalent of git add).
  3. Write a Commit Message: In the text box at the top of the Source Control panel, type a short, descriptive message about what you changed (e.g., "Fix login button color").
  4. Commit: Click the big blue Commit button (or the checkmark ✓ icon at the top).
  5. Optional (Push to GitHub): If you want to send this to GitHub, click the Sync Changes button that appears, or click the ... menu and select Pull, Push -> Push.

How to Commit in Visual Studio Community (2022 / 2026)

Microsoft has highly integrated Git into the core Visual Studio Community experience.

  1. Open the Git Changes Window: Go to the top menu and click View -> Git Changes (or press Ctrl+0, Ctrl+G).
  2. Stage Your Changes: In the Git Changes window, you will see your modified files under "Changes". Hover over a file and click the + (plus icon) to stage it.

    Note: If you just want to commit everything, you can skip staging and let VS do it automatically in the next step.

  3. Write a Commit Message: Type your description in the text box provided in the Git Changes window.
  4. Commit: Click the dropdown button that says Commit All or Commit Staged (depending on if you staged specific files).
  5. Optional (Push to GitHub): After committing, you will see an upward arrow icon (Push). Click it to push your changes to your remote GitHub repository.

5. How to Deploy a Website using GitHub Pages

Deploying a front-end project like PETOPIA or Cera takes only a few clicks and gives you a live, shareable URL for free. This is perfect for static sites (HTML, CSS, JavaScript) or built frameworks like React.

Step-by-Step Deployment:

  1. Prepare Your Repository: Ensure your project is pushed to a GitHub repository. Your main entry file must be named index.html and placed in the root folder.
  2. Go to Settings: Open your repository on GitHub and click on the Settings tab.
  3. Navigate to Pages: Scroll down the left sidebar and click on Pages (under the "Code and automation" section).
  4. Choose the Source: Under "Build and deployment," look for the "Source" dropdown and select Deploy from a branch.
  5. Select the Branch: In the Branch dropdown, select your main branch (usually main or master) and keep the root folder selected. Click Save.
  6. Wait for the Build: GitHub will automatically start building your site. After a few minutes, refresh the page, and you will see your live link at the top (usually formatted as https://[your-username].github.io/[repo-name]/).

6. How to Get and Use the GitHub Student Developer Pack

Applying with a Capital University email address unlocks the GitHub Student Developer Pack, which provides free access to premium developer tools, cloud hosting, and software licenses that usually cost hundreds of dollars.

How to Apply:

  1. Go to GitHub Education: Visit education.github.com/pack.
  2. Log In & Verify: Sign in with your GitHub account and click "Get your Pack."
  3. Submit Proof: Add your university email address. If prompted, you may need to upload a clear photo of your student ID, a current class schedule, or an enrollment verification letter.
  4. Wait for Approval: Applications are usually processed within a few days.

Top Benefits to Use:

  • GitHub Copilot Pro: Free access to the AI pair programmer directly inside your IDE.
  • Free Domains & Hosting: Use Namecheap (1-year free domain) and DigitalOcean ($200 credit) to host full-stack projects, databases, or APIs that GitHub Pages cannot support.
  • Pro IDEs: Free access to the entire JetBrains suite, which is excellent for heavy backend development.
