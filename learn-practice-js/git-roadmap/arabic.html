2. خارطة طريق تعلم الـ Git و GitHub

بعد ما تخلص الفيديو، امشي على الخطوات دي عشان تثبت المعلومة بالتطبيق العملي:

المرحلة الأولى: أساسيات الـ Local
  • المفاهيم: الفرق بين الـ Working Directory والـ Staging Area والـ Repository.
  • الأفعال: تعمل Init للحاجة، وتشوف الـ Status، وتعمل Stage للملفات، وتعمل أول Commits ليك.
  • الهدف: تبقى عارف تتابع مشروعك الشخصي على جهازك لوحدك.
المرحلة التانية: السفر عبر الزمن (تصليح الأخطاء)
  • المفاهيم: الـ HEAD والـ Commit Hashes.
  • الأفعال: تشوف الـ History، وتمسح التغييرات اللي لسه معملتلهاش Commit، وترجع لنسخة قديمة.
  • الهدف: تبقى واثق إنك مستحيل "تبوظ" الكود بتاعك نهائياً، دايماً فيه رجعة.
المرحلة التالتة: الـ Branching والـ Merging
  • المفاهيم: التطوير بالتوازي (Parallel Development).
  • الأفعال: تعمل Branch جديد لميزة معينة، وتنقل بين الفروع، وتدمج الميزات دي في الكود الأساسي، وتحل الـ Merge Conflicts.
  • الهدف: تطور ميزات جديدة بأمان من غير ما تبوظ النسخة "الشغالة" من الكود.
المرحلة الرابعة: الـ Remote (GitHub)
  • المفاهيم: الـ Local ضد الـ Remote، وطرق الـ Authentication (زي الـ SSH والتـوكنز).
  • الأفعال: تعمل Push للكود على GitHub، وتعمل Clone لمشاريع موجودة، وتسحب التحديثات (Pull).
  • الهدف: تحفظ نسخة من الكود بتاعك في الـ Cloud وتجهز نفسك للتعاون مع غيرك.
المرحلة الخامسة: التعاون والـ Open Source
  • المفاهيم: الـ GitHub Flow، والـ Forks، والـ Pull Requests.
  • الأفعال: تعمل Fork لمشروع حد تاني، وترفع تعديلاتك، وتطلب Pull Request عشان تتراجع وتتدمج.
  • الهدف: تساهم في مشاريع مفتوحة المصدر أو تشتغل مع فريق في شركة بكل سلاسة.

4. إزاي تعمل Commit بالواجهة الرسومية (من غير CMD)

مش لازم تستخدم الـ Terminal عشان تبقى شاطر في Git! الـ IDEs الحديثة فيها أدوات ممتازة.

طريقة الـ Commit في VS Code

  1. افتح الـ Source Control: من القائمة اللي على الشمال، دوس على الأيقونة اللي شبه الشجرة (أو اختصار Ctrl+Shift+G).
  2. اعمل Stage للتغييرات: اقف بالماوس على الملفات اللي غيرتها، ودوس على علامة الزائد +. ده معناه إنك بتعمل git add.
  3. اكتب رسالة الـ Commit: في الصندوق اللي فوق، اكتب رسالة قصيرة بتوصف أنت غيرت إيه (مثلاً: "صلحت لون زرار الدخول").
  4. اعمل Commit: دوس على الزرار الأزرق الكبير اللي مكتوب عليه Commit.
  5. خطوة زيادة (Push لـ GitHub): لو عايز تبعت التغييرات لـ GitHub، دوس على Sync Changes اللي هتظهر.

طريقة الـ Commit في Visual Studio Community

  1. افتح نافذة Git Changes: من المنيو اللي فوق اختار View وبعدين Git Changes (أو Ctrl+0, Ctrl+G).
  2. اعمل Stage: هتلاقي الملفات المتغيرة تحت كلمة "Changes"، دوس على علامة الـ + عشان تعملها Stage.
  3. اكتب الرسالة: اكتب الوصف بتاعك في مكان الكتابة المخصص.
  4. دوس Commit: اختار Commit All أو Commit Staged.
  5. خطوة زيادة (Push لـ GitHub): بعد الـ Commit هتلاقي سهم طالع لفوق (Push)، دوس عليه عشان ترفع شغلك.

5. إزاي ترفع موقعك على GitHub Pages

رفع مشاريع الـ Front-end زي PETOPIA أو Cera بياخد كذا ضغطة بس، وبيديك لينك شغال مجاناً.

خطوات الـ Deployment:

  1. جهز الـ Repo: اتأكد إن مشروعك مرفوع على GitHub، وإن الملف الأساسي اسمه index.html وفي الفولدر الرئيسي (Root).
  2. روح للـ Settings: افتح الـ Repo بتاعك على GitHub ودوس على Settings.
  3. ادخل على Pages: من المنيو اللي على الشمال، اختار Pages.
  4. اختار الـ Source: من تحت "Build and deployment"، اختار Deploy from a branch.
  5. حدد الـ Branch: اختار الـ branch الأساسي (غالباً main) ودوس Save.
  6. استنى شوية: GitHub هيبدأ يبني الموقع، بعد دقايق اعمل ريفريش وهتلاقي اللينك ظهرلك فوق.

6. إزاي تاخد الـ GitHub Student Developer Pack

لو قدمت بإيميل الجامعة، GitHub بيديك باقة فيها أدوات برمجة وسيرفرات بفلوس كتير جداً مجاناً ليك كطالب.

إزاي تقدم:

  1. افتح GitHub Education: زور الموقع education.github.com/pack.
  2. سجل دخول: ادخل بحسابك ودوس "Get your Pack".
  3. ارفع الإثبات: ضيف إيميل الجامعة، وممكن يطلبوا صورة للكارنيه أو جدول المحاضرات.
  4. استنى الموافقة: الموضوع بياخد كام يوم مراجعة.

أهم المميزات اللي هتاخدها:

  • GitHub Copilot Pro: مساعد الذكاء الاصطناعي مجاناً ليك جوه الـ IDE.
  • دومين واستضافة مجانية: من مواقع زي Namecheap و DigitalOcean عشان ترفع مشاريعك الكبيرة.
  • الـ IDEs الاحترافية: اشتراكات مجانية في برامج JetBrains القوية جداً.
